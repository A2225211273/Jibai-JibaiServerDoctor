## 🩺 这是什么？

**JibaiServerDoctor（服务器医生）** 是一个专门帮服主"看病"的插件。

服务器卡了，很多人第一反应是装一堆清理插件、优化插件，结果卡还是卡，因为**根本不知道问题出在哪**。服务器医生要解决的正是这一步——先诊断，再对症。

它会像医生做体检一样，帮你回答这些问题：

- 服务器为什么卡？
- 哪个世界压力最高？
- 哪些区块可能有问题？
- 哪些实体数量异常？
- 哪些插件在疯狂报错？
- 红石或漏斗是不是在偷偷拖后腿？
- 内存是不是一直居高不下？
- 现在服务器到底健不健康？
- 出问题了，我该把什么信息发给开发者

最后，它会给你一份**新手也能看懂的体检报告**，以及一个**能直接丢给AI的诊断包**。

> 💡 一句话总结：**别的插件帮你"清理"，服务器医生帮你"看懂"。**

## ✨ 核心亮点

### 1. 全核心通用，不挑食
纯 Bukkit/Spigot API 编写，不用 Paper 专属接口、不用 NMS、不用反射。
Bukkit / Spigot / Paper / Purpur 都能跑，已在 **Paper 1.21.8** 实测通过。

### 2. 0-100 健康评分，一眼看懂
把内存、实体、掉落物、区块、红石、漏斗、报错、tick 延迟等因素综合成一个分数：

| 分数 | 等级 | 含义 |
| --- | --- | --- |
| 90-100 | 🟢 优秀 | 状态很棒 |
| 70-89 | 🔵 正常 | 一切正常 |
| 50-69 | 🟡 需要关注 | 有点苗头了 |
| 30-49 | 🟠 存在风险 | 该处理了 |
| 0-29 | 🔴 严重异常 | 赶紧排查 |

### 3. 不只报问题，还教你怎么办
每一条风险都附带**原因**和**处理建议**，比如：

```
1. [严重] world 世界实体数量过高：共 5860 个实体
   建议：实体过多会显著增加每 tick 计算量，请用 /doctor inspect
        定位实体密集的区块，常见来源是刷怪塔、养殖场或掉落物堆积。
```

### 4. 一键生成体检报告
`/doctor report` 生成一份 Markdown 报告，包含服务器环境、内存、实体、区块、红石、漏斗、报错、评分和问题排名，存档随时看。

### 5. 诊断包 + AI 提示词，排障效率翻倍
`/doctor export` 把报告、错误摘要、环境信息、配置和一份 `ai-prompt.txt` 打包。
**把 `ai-prompt.txt` 的内容直接复制给 AI，就能得到针对性的优化建议。**

### 6. 精准定位到区块和玩家
- `/doctor inspect chunk`：看你脚下这个区块的实体、掉落物、漏斗、红石情况。
- `/doctor inspect player <玩家名>`：看某个玩家周围是不是卡顿源头。

### 7. 红石 / 漏斗风险监控（只提醒，不动手）
- 红石高频区块：按时间窗口统计，超阈值提醒。
- 漏斗密集区块：定时扫描，超阈值提醒。
- **重点：只记录和提醒，绝不破坏你的任何方块。**

### 8. 日志错误分析
自动读取 `logs/latest.log` 最近若干行，统计 ERROR / WARN / Exception，聚合同类报错并尝试认出是哪个插件在闹脾气。

### 9. 管理员实时告警
健康分过低、内存爆表、红石漏斗超标时，自动提醒在线管理员，还带冷却时间，不会刷屏。

### 10. 安全第一
不删数据、不破坏方块、不清理实体、不卸载区块、不上传文件，导出内容还会对 IP / 密码 / Token 自动脱敏。

## 📋 指令大全

主指令：`/doctor`　别名：`/jdoctor`、`/serverdoctor`

| 指令 | 作用 | 谁能用 |
| --- | --- | --- |
| `/doctor status` | 查看服务器当前健康状态 | 有 `status` 权限 |
| `/doctor report` | 生成 Markdown 体检报告 | 有 `report` 权限 |
| `/doctor export` | 导出诊断包（含 AI 提示词） | 有 `export` 权限 |
| `/doctor inspect chunk` | 检查你所在的区块（玩家专用） | 有 `inspect` 权限 |
| `/doctor inspect player <玩家名>` | 检查指定玩家附近区域 | 有 `inspect` 权限 |
| `/doctor risks` | 查看当前风险列表 | 有 `risks` 权限 |
| `/doctor errors` | 查看最近日志错误摘要 | 有 `errors` 权限 |
| `/doctor reload` | 重载配置文件 | 有 `reload` 权限 |
| `/doctor help` | 查看指令帮助 | 所有人 |

> ⌨️ 所有指令都支持 **Tab 键补全**，控制台也能执行大部分指令。

## 🚀 快速上手（3 步）

1. **安装**：把 `JibaiServerDoctor-1.0.0.jar` 丢进 `plugins/`，重启服务器。
2. **体检**：进游戏或在控制台输入 `/doctor status` 看当前状态，`/doctor report` 生成报告。
3. **排障**：发现卡顿就 `/doctor export`，把 `ai-prompt.txt` 复制给 AI，照建议优化。

## ❓ 常见问题

**Q：它会帮我自动清理卡顿源头吗？**
A：不会，也不应该。服务器医生的定位是"诊断"，动手清理请交给你自己或专门的清理插件——这样更安全，也不会误删你精心搭建的红石机器。

**Q：为什么 tick 延迟不是标准 TPS？**
A：为了全核心通用，它不依赖 Paper 的 TPS API，而是自己测量真实 tick 间隔算出近似延迟（理想值 50ms，越低越好）。

**Q：诊断包会泄露我的服务器隐私吗？**
A：不会。它不打包敏感配置文件，导出前还会对 IP、密码、Token 等自动打码。

**Q：能自定义提示语和颜色吗？**
A：可以。`messages.yml` 里所有文本都能改，支持 `<green>` 标签和 `&a` 颜色码。
